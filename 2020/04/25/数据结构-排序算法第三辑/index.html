归并排序与基数排序：

归并排序

归并排序（MERGE-SORT）是利用归并的思想实现的排序方法，该算法采用经典的分治（divide-and-conquer） 策略（分治法将问题分(divide)成一些小的问题然后递归求解，而治(conquer)的阶段则将分的阶段得到的各答案”修 补”在一起，即分而治之)。

再来看看治阶段，我们需要将两个已经有序的子序列合并成一个有序序列，比如上图中的最后一次合并，要将 [4,5,7,8]和[1,2,3,6]两个已经有序的子序列，合并为最终序列[1,2,3,4,5,6,7,8]，来看下实现步骤

代码如下：（思路分析）

public class MergeSortDemo {

//归并排序

 public static void main(String args[]) {



     /*
     基本思想和思路分析：


     分：归并排序就是把一个数组的元素先分成两个部分，假设mid1= （0+arr.length-1）/2左边是从 0 到mid，右边是从mid+1到数组末尾，按照

     该种方法，再将分为两个部分的数组元素，左边：（左边： mid2=(0+mid1)/2,0到mid2,右边mid2+1到mid1 ）,右边：同样道理...依此类推来分直到每一个部分的元素被分为单个元素。

      比如 ： 84571362

      8457  1362

   84  57   13  62

 8 4   5 7   1 3   6 2

 治：先从最细分的开始合并，


 分应该是一个递归的过程，就是细分到最细了就从栈的上往下合并，也就是从最细的到完整的。

 单个元素的合并，哪几个单元素是之前被一个部分拆下来的，就把它们按照有序的方式合并成原来的部分。逐步合并成完整的，每一次合并都要把合并后的
 序列放到一个新的数组中，最后合并完了再把这个中间数组拷贝到原数组。

 比如：  48   57   13   26

          4578   1236

           12345678


     */


     int [] arr={ 8,4,5,7,1,3,6,2};
     int [] temp=new int[arr.length];

     System.out.println("前"+ Arrays.toString(arr));
     breakArr(arr,0,arr.length-1,temp);
     System.out.println("后"+Arrays.toString(arr));





 }

 //分解数组
 public  static void breakArr(int [] arr,int left,int right,int [] temp){



     //什么时候不再分解了
     if(left<right){

         int mid=(left+right)/2;
         //向左递归分解
         breakArr(arr,left,mid,temp);

         //向右递归分解

         breakArr(arr,mid+1,right,temp);

         //栈顶进行合并，就是左递归一直到最后不能递归了，就执行分解操作，同样道理

         merge(arr,temp,left,right);



         //程序是这样执行的，首先从开始进来分解完左边，再去分解左边，再去分解左边，直到左边不能再被分解了就回退到上一级别（3）
         // 的部分向右分解进入下一级别，右边分解完（分一次）就不能分解了，然后回退上一级别(也就是3)，3这个级别再执行合并。然后3这个级别
         //都执行完了就回退到上一级别（2），2呢，就继续执行它的向右分解的代码，直到。。。。。执行回退到该级别，就进行合并..

     }




 }


 //合并的方法
 public  static void merge(int arr[],int [] temp,int left,int right){


     //arr是原数组，temp是临时数组，left是左指针，就是最左边的索引，right是右指针，就是最右边的索引


     int i=left;//i是会移动的，是左边的可移动指针，初始值为左边的索引
     int mid=(left+right)/2;  //中间
     int j=mid+1;//j是可移动的右边的指针，初始值为右边的索引
     int t=0;//t是临时数组的索引


     //合并的时候，比如拿最后一次合并来说，有两个有序的部分， 先拿左边的部分的第一个元素与右边的第一个元素相比，如果左边第一个元素比右边
     //第一个元素小，那么就把左边第一个元素移到临时数组中，然后左边的元素移到下一个，继续和右边第一个元素比较，反之则把右边第一个元素移到临时数组,右边的
     //元素右移，和左边第一个元素比较。。。就这样比下去，直到有一个部分的元素都比完了，就把另外一个部分的剩余的元素按照索引填进临时数组里。

     while (i<=mid && j<=right){

         if(arr[i]<=arr[j]){
             //左边的元素小于右边的
             temp[t]=arr[i];
             i++;
             t++;

         }
         else{

             //左边的元素大于右边的

             temp[t]=arr[j];
             j++;
             t++;

         }

     }

     //循环结束，如果还有某个部分的元素剩余

     //左边的元素有剩余
     while (i<=mid){

         temp[t]=arr[i];
         i++;
         t++;

     }

     //右边有元素剩余
     while (j<=right){

         temp[t]=arr[j];
         j++;
         t++;
     }


     //现在是把临时数组的元素拷贝到原数组

     //每一次合并完就要拷贝进原数组。每一次的临时数组都会从0开始覆盖原数组，直到最后合并成有序的了，
     // 就从0开始覆盖原数组，就完事了

     int tempLeft=left; //从left开始。这样一个部分的左右两个部分合并才不会造成合并了左后进去原数组了，但是合并了右部分后进入原数组把左的覆盖掉。
     t=0;
     while (tempLeft<=right){

         arr[tempLeft]=temp[t];

         t++;
         tempLeft++;

     }






 }



}

定义10个桶，每一个桶是一个一维数组，
数组元素的最大位数是n（若n=3）
第一次把数组中的元素的个位数一一拿出来，是多少就放在相应下标的桶里。
放完后，就把桶中的数据取出按顺序取出来放在原数组里面。
清空桶的数据
第二次就把数组中的元素的10位数一一拿出来，是多少就放在相应下标的桶里
再把桶中的数据按顺序拿出来放在原数组。
清空桶的数据
第三次就把百位数拿出来。。。还是这样操作
 */

 public static  void radixSort(int [] arr,int [][]bucket,int [] bucketElementCount ){

     //整合一下

     //判断数组中的最大位数的长度是多少，这样才可以知道需要进行多少次才到达有序。数组中最大数的位数为多少，就要进行多少次的排序操作

     //定义一个最大值,假设是数组的第一个数组

     int max= arr[0];
     for(int i=0;i<arr.length;i++){

         if(arr[i]>max){

             max=arr[i];
         }

     }

     //求最大值的长度
     int maxLength=(max+"").length();

     int count=0;
     //那就循环maxLength次

     for(int i=0,n=1;i<maxLength;i++,n*=10){

       count++;

         //遍历数组
         for (int j=0;j<arr.length;j++){

             int digits=arr[j]/n%10;  //分别取模10的倍数
             bucket[digits][bucketElementCount[digits]]=arr[j]; //bucket[每一轮的各个个/十/百...位数][记录每一轮中的各个个/十/百..位数的个数]
             bucketElementCount[digits]++;
         }

         //遍历桶，放进原数组
         int index=0;
         for(int k=0;k<bucketElementCount.length;k++){


             //bucketElementCount[?] ?表示某一轮的个/百/..位数，总体表示有多少个
             //可能存在有的桶没有值，所以要做判断,因为没有值的桶，bucketElementCount这个数组不会记录进去
             if(bucketElementCount[k]!=0){

                 //说明下标为k的桶是有数据的,只要bucketElementCount[k]有数据。说明bucket[k][？]是有数据的 ，？就是bucketElementCount[k]

                 for(int l=0;l<bucketElementCount[k];l++){

                     //把桶的值放进数组里面
                     arr[index]=bucket[k][l];
                     index++;

                 }

             }
             //清空桶的数据个数,防止对下一次造成影响

            bucketElementCount[k]=0;  //!!!不然的话就会把原来的数据放进数组里面.



         }
System.out.println("第"+count+Arrays.toString(arr));

     }




 }


}

基数排序的说明:

  1. 基数排序是对传统桶排序的扩展，速度很快。

  2. 基数排序是经典的空间换时间的方式，占用内存很大, 当对海量数据排序时，容易造成 OutOfMemoryError 。

  3. 基数排序时稳定的。[注:假定在待排序的记录序列中，存在多个具有相同的关键字的记录，若经过排序，这些 记录的相对次序保持不变，即在原序列中，r[i]=r[j]，且 r[i]在 r[j]之前，而在排序后的序列中，r[i]仍在 r[j]之前， 则称这种排序算法是稳定的；否则称为不稳定的]

  4. 有负数的数组，我们一般不用基数排序来进行排序
