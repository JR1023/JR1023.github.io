贪心算法的应用：

贪心算法 应用场景-集合覆盖问题

假设存在下面需要付费的广播台，以及广播台信号可以覆盖的地区。 如何选择最少的广播台，让所有的地区 都可以接收到信号

  1. 贪婪算法(贪心算法)是指在对问题进行求解时，在每一步选择中都采取最好或者最优(即最有利)的选择，从而 希望能够导致结果是最好或者最优的算法

  2. 贪婪算法所得到的结果不一定是最优的结果(有时候会是最优解)，但是都是相对近似(接近)最优解的结果

使用贪婪算法，效率高:

  1. 目前并没有算法可以快速计算得到准备的值， 使用贪婪算法，则可以得到非常接近的解，并且效率高。选择策略上，因为需要覆盖全部地区的最小集合:
  2. 遍历所有的广播电台, 找到一个覆盖了最多未覆盖的地区的电台(此电台可能包含一些已覆盖的地区，但没有关 系）
  3. 将这个电台加入到一个集合中(比如 ArrayList), 想办法把该电台覆盖的地区在下次比较时去掉。
  4. 重复第 1 步直到覆盖了全部的地区

代码实现：

    public class GreedyAlgorithm {


 public static void main(String args[]) {

     //贪心算法：每次寻找最优解

     //问题:
     /*
     有付费广播覆盖：

          k1      北京，上海，天津
          k2      广州，北京，深圳
          k3      成都，上海，杭州
          k4      上海，天津
          k5      杭州，大连

          选择最少的广播电台，覆盖这里的所有地区
      */

     /*
     思路分析：首先需要一个把所有的地区全都放进一个集合里面，方便进行对照在选择一个电台后哪些地区已经被覆盖，
     然后可以把这些地区从集合中干掉，直到最后没有了就说明ok了

     用一个hashMap<String,hashSet<String>>来存放  "k1      北京，上海，天津"这种结构，

     需要遍历每一个广播的地区，然后呢把地区放在一个临时的Arrlist集合里面,去与所有地区的集合求交集，把结果赋到这个临时集合里面
      一个for循环遍历每一个广播的地区，定义一个maxKey，进来后首先指向遍历的第一个广播，然后判断下一个广播覆盖的地区是否比
      maxKey指向的多，多的话maxKey就指向新的,要是小于或者等于就不用改变指向

      */

     HashSet<String> hashSet1=new HashSet<>();
     HashSet<String> hashSet2=new HashSet<>();
     HashSet<String> hashSet3=new HashSet<>();
     HashSet<String> hashSet4=new HashSet<>();
     HashSet<String> hashSet5=new HashSet<>();

     hashSet1.add("北京");
     hashSet1.add("上海");
     hashSet1.add("天津");

     hashSet2.add("广州");
     hashSet2.add("北京");
     hashSet2.add("深圳");

     hashSet3.add("成都");
     hashSet3.add("上海");
     hashSet3.add("杭州");

     hashSet4.add("上海");
     hashSet4.add("天津");

     hashSet5.add("杭州");
     hashSet5.add("大连");

     HashMap<String, HashSet<String>> map=new HashMap<>();

     map.put("k1",hashSet1);
     map.put("k2",hashSet2);
     map.put("k3",hashSet3);
     map.put("k4",hashSet4);
     map.put("k5",hashSet5);


     //需要一个集合来存放全部的地区

     List<String>  allAreas  =new ArrayList<>();

     allAreas.add("北京");
     allAreas.add("上海");
     allAreas.add("天津");
     allAreas.add("广州");
     allAreas.add("深圳");
     allAreas.add("成都");
     allAreas.add("杭州");
     allAreas.add("大连");


     greedyAlgorithm(map,allAreas);
 }

 public static void  greedyAlgorithm(Map<String,HashSet<String>> map, List<String> allAreas){

     //定义一个临时的集合，存放该广播对应的地区
     List<String> temp=new ArrayList<>();

     //定义一个存放结果集的集合
     List<String> selected=new ArrayList<>();

       String maxKey=null;
       int MaxTempSize=0;
     //遍历传进来的map集合,得到每一个广播
     while (allAreas.size()>0){
         //因为每次添加一个广播就要删掉allAreas集合中该广播对应的覆盖地区，最后这个集合长度为0了就完了

         //每遍历完一次全部的广播得到一个广播结果集的时候，就要把maxKey置空，
         // 让它下次遍历全部广播的时候maxKey先指向第一个
         maxKey=null;
     for(String key:map.keySet()) {

         //每次遍历到一个广播就清空以前的temp集合

         temp.clear();

         //得到该广播对应的覆盖地区
         HashSet<String> hashSet = map.get(key);

          for(String area: hashSet){

              temp.add(area);

          }

          temp.retainAll(allAreas);//求交集，把交集的值重新赋给temp集合

         if(temp.size()>MaxTempSize){

             MaxTempSize=temp.size();

         }

          //判断temp的长度

         if(temp.size()>0 && (maxKey==null || temp.size()>MaxTempSize)){

             //把key赋为maxKey
             maxKey=key;
         }


     }

     if(maxKey!=null){

         selected.add(maxKey);

         allAreas.removeAll(map.get(maxKey));
     }



     }






     //输出结果集

     for(String s:selected){

         System.out.println("放入广播"+s);
     }

 }


}
